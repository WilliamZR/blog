Logical Representations of Sentence Meaning

语言学中有一些复杂的人物，比如制作菜单，菜单中的各种材料都有自身的common sense。对于一个人来讲在看到菜单上的菜品的时候就可以意识到自己在看的是什么菜，而对于机器来说这样的任务比较难的。

在机器理解这些含义的过程中，需要knowledge base的辅助

我们在这个章节的做出的假设是：语言学的表达可以通过某种处理，转化成和knowledge base同样的形式，这个过程被称之为semantic parsing or semantic analysis

Computational Desiderata for Representaions

Verifiability
  • A system’s ability to compare the state of affairs described by a representation to the state of a world modeled in a knowledge base.
Unambiguous Representations

句子本身可以是ambiguous的，但是meaning representation不应当有歧义

区分概念：

  • ambiguity

    I want to eat someplace near New York

  • vagueness (leave some staff unspecified)

I want to eat some Italian food

Canonical Form
  • Distinct inputs meaning the same thing should have the same meaning representation
Inference and Variables

Can vegetarians eat at Maharani?

Common sense: what vegetarians eat and what Maharani serves

Inference: draw conclusion based on meaning representation and background knowledge.

I’d like to find a restaurant where I can get vegetarian food.

This request a restaurant not named $Serves(x, VegetarianFood)$

Variable : Matching succeeds only if $x$ can be replaced with some object in knowledge base.

Expressiveness
  • Expressive enough to handle a wide range of subject mattter

Model Theoretic Semantics

  • Vocabulary

    • Non-logical open-ended set of names for the objects, properties and relations we are trying to represent

      Each element have a denotation in the model.

    • Logical closed set of symbols, operators, quantifiers, links…

  • Domain: set of objects being represented

    • Objects: elements of the domain
    • Properties: set of elements of the domain
    • Relations: set of tuples of elements of the domain

  • Interpretation : mapping from meaning representation to denotation

    • 尽管对于一个property或者relation来说，这样的推理是非常直接的，但是在实际的情况中，句子的组成可能是非常复杂的，需要不同意义之间的组合。
    • Truth-conditional Semantics: a methof for determining the truc of a complex expression

First Order Logic

Basic Elements

Term

  • Constant: specific objects in the world being described. Contant refer to exactly one object while objects can have multiple constantss that refer to them

  • Functions：similar as single argumnet predicates, but refer to object

    $LocationOf(Frasca)$

  • Variables: Single lower-case letters, let us make assertions and draw inferences about
    objects without having to make reference to any particular named object.

Variables and Quantifiers

  • quantifier: map a variable to particular objects $\exists$ or generically all objects in a collection $\forall$

    $∀xVegetarianRestaurant(x) =⇒ Serves(x,VegetarianFood)$

    对于这样的一个表达式要成立，需要将所有的object对x进行替换，只有所有object都成立的时候断言才为真。而在不是VegetarianRestaurant的餐厅替换x的时候，蕴涵断言式仍为真

Lambda Notation

  • We can apply them to logical terms to yield new FOL expressions where the formal paramerter variables are bound to the specified terms.

    $\lambda .P(x)(A) \rightarrow P(A)$

The Semantic of FOL

Inference

  • Modus ponens a form of inference(if-then reasoning)

    If left-hand side is true, then right-hand side is true.

  • Two ways of practical use

    • Forward: Once new facts are added to knowledge base, all applicable implication rules are applied. All inference is performed in advance

    • Backward:recursively backward chaining

    • Remark: both forward and backward chaining are not complete, there are inferences counnot be found by them. Resolution is sound and complete, but nomr computationally expensive.

Event and State Representations

  • States are conditions, or properties, that remain unchanged over an extended period of time,

  • Events denote changes in some state of affairs.

  • Predicates in FOL have fixed number of arguments (fixed arity), we introduce an event variable

    $∃e Eating(e) ∧ Eater(e,Speaker)∧Eaten(e,TurkeySandwich)
    ∧Location(e,Desk)$
